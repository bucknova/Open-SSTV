# v0.3 Template Compositor — Design Plan

**Status**: Draft, not yet implemented. Planned for v0.3, after v0.2.7 ships.
**Last updated**: 2026-04-17

## 1. Overview

v0.2.x treats templates as three plain strings in `AppConfig` (`template_cq`,
`template_exchange`, `template_73`). Each string is stamped onto the TX image
via a single text overlay. This is functional but primitive — no multi-block
layouts, no per-template styling, no placeholder tokens for dynamic fields,
no way for users to author templates in a GUI, no visual template picker.

v0.3 replaces this with a **layered template compositor** modeled on MMSSTV
but built with a modern data model. The user authors templates as a stack of
layers (photo, RX slot, text, rects, gradients, patterns), each with
anchor-based positioning and full styling (stroke, shadow, slashed-zero
handling). Templates carry placeholder tokens (`%c`, `{callsign}`, etc.) that
resolve at TX time from AppConfig + per-QSO state + live context. A visual
gallery in the TX panel shows live-rendered thumbnails at the current mode's
aspect ratio, letting users pick by appearance rather than by name.

## 2. Goals / non-goals

### Goals
- Feature parity with MMSSTV's template system, with modern polish.
- Live composite preview in TX panel, with thumbnail gallery for selection.
- Extensible layer model — new layer types without breaking existing
  templates (schema_version gating).
- Shareable templates (TOML single-file, or ZIP bundle with assets).
- Aspect-ratio aware: templates render correctly across all SSTV modes
  (320×240 Robot 36, 320×256 Scottie/Martin, 640×496 PD-120).
- Full round-trip: editor → TOML → renderer → SSTV encoder, fully
  unit-testable as pure-function transforms.
- Fonts shipped with the app so shared templates render identically across
  machines.

### Non-goals for v0.3.0
- Animation / multi-frame templates.
- Scripted conditional logic.
- Template-rendered-as-PNG export for use outside SSTV.
- QSO logging (ADIF export) — parked for v0.4.
- CW ID customization per-template — stays in its own config slot.
- Web-hosted fonts / download-on-demand — offline-first only.

## 3. Architecture overview

```
┌──────────────────────────────────────────────────────────┐
│  TX Panel                                                │
│  ┌────────────────────────────────────────────────────┐  │
│  │      Main composite preview (live)                 │  │
│  └────────────────────────────────────────────────────┘  │
│  ┌────────────────────────────────────────────────────┐  │
│  │      Template Gallery (thumbs, role filter)        │  │
│  └────────────────────────────────────────────────────┘  │
│  ┌────────────────────────────────────────────────────┐  │
│  │      QSO State widget (ToCall / RST / Name)        │  │
│  └────────────────────────────────────────────────────┘  │
└──────────────────────────────────────────────────────────┘
                        │
                        ▼
   ┌────────────────────────────────────┐
   │      TemplateRenderer (pure)       │
   │                                    │
   │   (template, qso_state, cfg, ctx)  │
   │              ↓                     │
   │   resolve tokens → absolute coords │
   │              ↓                     │
   │   rasterize layers bottom→top      │
   │              ↓                     │
   │   mode-aware resize                │
   │              ↓                     │
   │    PIL.Image (RGB, mode-sized)     │
   └────────────────────────────────────┘
                        │
                        ▼
            SSTV encoder (existing)
```

Renderer is a pure function — no Qt imports, no file I/O, no side effects.
Same transform runs for live preview, gallery thumbnails, and final TX.

## 4. Data model

### 4.1 Template
```python
@dataclass
class Template:
    name: str
    role: Literal["cq", "reply", "closing", "custom"]
    reference_frame: tuple[int, int]  # design canvas, default (320, 256)
    schema_version: int               # currently 1
    description: str                  # user-facing note
    layers: list[Layer]               # ordered bottom → top
```

### 4.2 Layer (base)
Common fields shared by every layer type:
```python
@dataclass
class LayerBase:
    id: str
    name: str
    visible: bool = True
    opacity: float = 1.0          # 0.0 – 1.0
    anchor: Anchor                # TL|TC|TR|CL|C|CR|BL|BC|BR|FILL
    offset_x_pct: float = 0.0     # relative to anchor point
    offset_y_pct: float = 0.0
    width_pct: float | None = None    # optional sizing
    height_pct: float | None = None
    rotation_deg: float = 0.0     # optional, default 0
```

### 4.3 Layer types (discriminated union on `type` field)

1. **PhotoLayer** — user-selected TX photo. `fit`: `contain | cover | stretch`.
   Optional crop rectangle (future).
2. **RxImageLayer** — most-recently-received image from RxPanel state.
   Expires on `transmission_complete`. Falls back to transparent when absent.
3. **StationImageLayer** — fixed image from Settings (QSL card, station photo).
   Path stored relative to `{user_config_dir}/open_sstv/assets/`.
4. **TextLayer** — see §4.4.
5. **RectLayer** — solid or semi-transparent rectangle. `fill: RGBA`.
   Used for banner strips.
6. **GradientLayer** — two-stop linear gradient. `from_color, to_color, angle_deg`.
7. **PatternLayer** — tiled pattern. Ships with a small built-in pattern
   library (checkered, diagonal stripes, dots). `pattern_id, tint: RGBA`.

### 4.4 TextLayer style
```python
@dataclass
class TextLayer(LayerBase):
    type: Literal["text"] = "text"
    text_raw: str                       # with tokens: "CQ de %c"
    font_family: str                    # "DejaVu Sans Bold"
    font_size_pct: float                # % of frame height
    weight: Literal["regular", "bold"] = "regular"
    italic: bool = False
    fill: RGBA
    stroke: StrokeSpec | None = None    # {color, width_px}
    shadow: ShadowSpec | None = None    # {offset_x, offset_y, color, blur_px}
    align: Literal["left", "center", "right"] = "left"
    line_height_mult: float = 1.0
    slashed_zero: bool = True           # auto-render 0→Ø in callsign tokens
    date_format: str = "%Y-%m-%d"       # strftime, used if text contains %d
    time_format: str = "%H:%M"          # strftime, used if text contains %t
```

Image 2's cyan ghost behind "CQ de WØAEZ" = single TextLayer with a
`ShadowSpec`. Image 1's red-with-white-outline = TextLayer with a `StrokeSpec`.

## 5. Token resolution

### 5.1 Supported tokens

Both `%x` (MMSSTV-style) and `{name}` (named) syntaxes resolve to the same
values. Documented cheat sheet below:

| `%` form | Named form | Source | Notes |
|---|---|---|---|
| `%c` | `{callsign}` | AppConfig.callsign | own callsign |
| `%g` | `{grid}` | AppConfig.grid | own grid locator |
| `%n` | `{name}` | AppConfig.op_name | own op name |
| `%o` | `{tocall}` | QSOState.tocall | other station callsign |
| `%r` | `{rst}` | QSOState.rst | signal report (default `595` for SSTV) |
| `%name_o` | `{tocallname}` | QSOState.tocall_name | other op's name |
| `%note` | `{note}` | QSOState.note | free-form note |
| `%m` | `{mode}` | TXContext.mode.display_name | "Scottie S1" |
| `%d` | `{date}` | UTC now, strftime layer.date_format | |
| `%t` | `{time}` | UTC now, strftime layer.time_format | |
| `%f` | `{freq}` | RigState.frequency_hz | blank if no rig |
| `%b` | `{band}` | derived from freq | blank if no rig |
| `%q` | `{qso_serial}` | QSOState.serial | incrementing |
| `%v` | `{version}` | `open_sstv.__version__` | |
| `%%` | — | literal `%` | |

Unknown tokens render as-is (forward-compat).

### 5.2 Parser

Two-pass regex resolver in `open_sstv/templates/tokens.py`:
1. First pass: `%([a-z_]+)` — matches `%c`, `%name_o`, `%%`.
2. Second pass: `\{([a-z_]+)\}` — matches `{callsign}`, `{tocall}`.
Both passes use the same resolution table; order chosen so that `%%` literals
process before `{}` expansion (avoiding accidental interaction).

Callsign-valued tokens (`%c`, `%o`, etc.) apply the `slashed_zero` transform
when the containing TextLayer has `slashed_zero=True`: any `0` in the
resolved token value becomes `Ø`. Applied only to callsign tokens, never to
RST or grid locators.

### 5.3 Template authoring note
Authors **can** mix syntaxes in the same template but we don't advertise
that. Default starter templates use `%c` style for MMSSTV muscle-memory
familiarity.

## 6. Rendering pipeline

```python
def render_template(
    template: Template,
    qso_state: QSOState,
    app_config: AppConfig,
    tx_context: TXContext,
) -> PIL.Image.Image:
    resolved = resolve_tokens(template, qso_state, app_config, tx_context)
    target_size = tx_context.mode.frame_size  # e.g. (320, 240) for Robot 36
    canvas = PIL.Image.new("RGBA", target_size, (0, 0, 0, 0))
    for layer in resolved.layers:
        if not layer.visible:
            continue
        rasterize_layer(layer, canvas, target_size)
    return canvas.convert("RGB")
```

Per-layer rasterization uses Pillow:
- `ImageDraw.text(..., stroke_width=N, stroke_fill=...)` (≥8.0) for strokes.
- Shadows: prior `.text()` pass, offset, optionally `ImageFilter.GaussianBlur`.
- Rects: `ImageDraw.rectangle` or `Image.paste` for alpha-blended fills.
- Gradients: numpy gradient array → `Image.fromarray` → paste.
- Patterns: pre-loaded PIL image tiled via `Image.paste` in a loop.
- Photo/RX/Station image layers: `Image.resize` then `Image.paste` with mask
  computed from `fit` mode.

Pure function, easily unit-tested with golden-image tests.

### 7.1 Placement (TX panel)

```
┌─────────────────────────────────────────────────────────┐
│        [ Big main preview / active composite ]          │
├─────────────────────────────────────────────────────────┤
│  Template Gallery   [Role: CQ|Reply|73|All]  [+ New…]   │
│  ┌────┐ ┌────┐ ┌────┐ ┌────┐ ┌────┐ ┌────┐ ┌────┐      │
│  │ ✓  │ │    │ │    │ │    │ │    │ │    │ │    │  →   │
│  │thmb│ │thmb│ │thmb│ │thmb│ │thmb│ │thmb│ │thmb│      │
│  └────┘ └────┘ └────┘ └────┘ └────┘ └────┘ └────┘      │
│  CQSSTV  CQ de  ToCall Reply  73     Custom...          │
├─────────────────────────────────────────────────────────┤
│ QSO State: ToCall [W0XYZ] RST [595] Name [..]  Clear   │
├─────────────────────────────────────────────────────────┤
│ [ Transmit ] [ Stop ] [ Mode: Scottie S1 ▼ ]           │
└─────────────────────────────────────────────────────────┘
```

### 7.2 Thumbnail spec
- Base size: 140×112 px at 5:4. Reshapes per mode (140×105 for Robot 36 4:3).
- Content: live composite = photo + template + current QSO state + current
  mode dimensions. Truly WYSIWYG.
- Badges: corner chip showing role (CQ/Reply/73/Custom), checkmark for
  selected template.
- Caption: template name below thumbnail.
- Hover tooltip: larger preview (~320×256) + template description.

### 7.3 Interactions
| Gesture | Behavior |
|---|---|
| Left-click | Select + apply to main preview |
| Double-click | Open template in editor |
| Right-click | Context menu: Edit / Duplicate / Rename / Delete / Export / Set as default for role |
| Drag | Reorder within gallery |
| Drag to trash | Delete (with confirm) |
| "+ New…" | Open blank template in editor |

### 7.4 Role filter
Segmented control above the strip: `[All | CQ | Reply | 73 | Custom]`.
Defaults to *All*. Helps manage large custom template libraries.

### 7.5 Live-update triggers
Thumbnails re-render when:
- New photo loaded → all thumbs re-render with new photo.
- SSTV mode changed → aspect ratio shifts → all thumbs re-render.
- QSO state edited (ToCall/RST) → thumbs re-render (debounced).
- Template saved in editor → only affected thumb re-renders.

### 7.6 Performance strategy

Multi-tier cache keyed on independent invalidation axes:

- **Slow cache** `(template_id, photo_hash, mode)` — caches photo + all
  layers whose text resolves from AppConfig only (e.g. own-call banner,
  version banner). Invalidates on photo change or mode change (rare).
- **Fast overlay** — layers containing `%o`/`%r`/`{tocall}`/`{rst}` tokens
  re-rasterize fresh per keystroke on top of the cached base. Debounced
  300ms after last keypress.

With 10 templates visible, typing in ToCall re-rasterizes only the 2-3 text
layers per template containing QSO tokens. Roughly 20-30 `ImageDraw.text()`
calls per debounced update — trivial.

Additional mitigations:
- Async render via `QThreadPool` + `QRunnable`; placeholder grey tile while
  real thumb is in flight.
- Progressive: currently-selected thumb first, then visible, then off-screen
  as scrolled into view.
- Downscale photo once into shared 140-wide buffer, reuse across thumbs.

Phased rollout:
- **v0.3.0**: synchronous render on GUI thread. Fine for ≤10 templates.
- **v0.3.1**: async render pipeline + cache tiers + debouncing.

### 7.7 No-photo placeholder
Three options shipped in Settings dropdown:
1. **Open-SSTV test card** (default) — branded, muted gradient background,
   logo bottom-right, "TEST IMAGE — NO PHOTO LOADED" watermark.
2. **SMPTE color bars** — classic broadcast test pattern.
3. **Solid 50% grey** — minimalist.

### 7.8 Mode-mismatch indicator
If a template was authored at one aspect ratio and text content clips into
the safe-area margin of the current mode, the thumbnail shows a small ⚠
badge. Clicking the badge opens the editor with safe-area guides overlaid.

## 9. QSO State widget

Collapsible section in TX panel:
- **ToCall** — uppercased-on-type (same widget as first-launch dialog).
- **RST** — combo box, default `595`, history of recent values.
- **Name** — other op's name (optional).
- **Note** — free-form (optional).
- **Template role** — implicit, driven by which template is selected; no
  explicit radio needed here (role filter lives on gallery).
- **[Clear QSO]** button — wipes ToCall, RST, Name, Note.

State persists across TXs within a single QSO (operators often send multiple
pictures). Clears only on:
- Explicit [Clear QSO] click.
- ToCall field changes to a different callsign (heuristic: "different QSO").

RX image slot clears on `transmission_complete` regardless — the decoded
image remains in the gallery for viewing.

### 10.3 Schema versioning
- `schema_version = 1` for all v0.3 templates.
- Loader refuses templates with `schema_version > N_supported` with a
  friendly "this template is from a newer version of Open-SSTV" message.
- Unknown fields within a layer are ignored silently (forward-compat for
  future layer-type properties).

## 11. Resolution / aspect-ratio handling

SSTV modes vary in both size and aspect ratio. The compositor **must**
produce valid output for any of them from a single template definition.

**Strategy: normalized coordinates + safe-area discipline.**

- All layer positions and sizes stored as **% of frame**, never pixels.
- A template is just a recipe; the renderer takes whatever target size the
  current mode demands.
- Reference design frame = 320×256 (5:4). Editor opens here by default.
- **Mode preview selector** in editor: dropdown of every SSTV mode. Switch
  to Robot 36 → editor canvas reshapes to 320×240, template re-renders.
- **Safe-area guides** (toggleable in editor): dashed overlay showing the
  inner region that's safe across all supported modes. Text and critical
  content belong inside the safe area.
- **Mode-mismatch badge** on gallery thumbs: if a template's text content
  clips the current mode's safe area, badge appears on the thumbnail.

### Supported modes and frame sizes
| Mode | Frame size | Aspect |
|---|---|---|
| Robot 36 | 320×240 | 4:3 |
| Scottie S1 / S2 | 320×256 | 5:4 |
| Martin M1 / M2 | 320×256 | 5:4 |
| PD-50 / PD-90 | 320×256 | 5:4 |
| PD-120 / PD-180 | 640×496 | ≈5:4 |
| PD-240 / PD-290 | 640×496 | ≈5:4 |

Safe area = intersection of all aspect ratios, roughly the inner 90%
vertically of the 5:4 reference frame.

## 12. Fonts

### 12.1 Shipped Tier 1 (always available, default set)

All three open/free, GPL-compatible:

| Font | License | Role |
|---|---|---|
| DejaVu Sans Bold | Bitstream Vera (permissive) | Workhorse default, banner text, callsigns |
| Inter Bold | SIL Open Font License 1.1 | Modern alternative, screen-optimized |
| Press Start 2P | SIL Open Font License 1.1 | Pixel/bitmap font for retro templates (image 5 "73") |

Total size added to installer: ~2 MB.

### 12.2 User-imported fonts
Settings → Fonts → "Import font…" dialog.
- Accepts TTF / OTF.
- Copies into `{user_config_dir}/open_sstv/fonts/`.
- Parses family name via `fontTools` for template references.
- Template renderer looks in: (a) user fonts dir, (b) shipped fonts dir, in
  that order.

### 12.3 Missing-font handling
If a template references a font the current install doesn't have:
- Render falls back to DejaVu Sans Bold for the affected layer.
- Editor shows a `[missing: Bebas Neue]` badge on the layer in the layer list.
- Export bundle option: "Include referenced fonts" (when allowed by license).

### 12.4 Why we ship fonts (not just rely on system fonts)
- Shared templates must render identically on mac/windows/linux.
- SSTV is narrowband — fonts must be chosen and validated at target
  resolution (≤640 pixels wide).
- Slashed-zero availability varies wildly by system font; shipping
  guarantees it.

### 12.5 Parked font decisions
- Tier 2 add-on pack (Bebas Neue, Fira Mono, VCR OSD Mono) — decide during
  v0.3.0 polish phase.
- Font size discipline for Press Start 2P (multiples of base size) — will
  document as a lint warning in editor during v0.3.1.

## 13. Migration from v0.2.x

On first v0.3 launch:

1. If `{config_dir}/templates/` exists and is non-empty → no migration, user
   already has v0.3 templates.
2. Otherwise, check legacy strings `template_cq`, `template_exchange`,
   `template_73` in `AppConfig`:
   - If any differs from the v0.2 default → auto-generate corresponding
     TOML template with a single bottom-centered TextLayer carrying that
     string (white fill, black stroke, DejaVu Sans Bold, sensible size).
   - If all match defaults → skip migration, install starter pack instead.
3. Starter pack: five templates mirroring MMSSTV screenshots (CQSSTV,
   CQ de WØAEZ, Reply/ToCall 595, Reply Simple, 73).
4. Legacy `template_*` strings remain in `AppConfig` read-only for rollback
   safety; v0.3 code never writes to them.

## 14. Import / export

### 14.1 Single template
- Export: copies `.toml` to user-chosen location. If the template references
  StationImage paths, prompts with "Template references external images.
  Export as bundle instead?"
- Import: drops `.toml` into `{config_dir}/templates/`. Validates
  `schema_version`. Refuses newer-version templates with a friendly error.

### 14.2 Bundle
ZIP containing:
```
manifest.toml          # name, author, description, schema_version
templates/
  template_a.toml
  template_b.toml
assets/
  qsl_card.png
  station_photo.jpg
fonts/                 # optional, only if author chose to include
  CustomFont.ttf
```
- Export: user picks one or more templates + "Include referenced assets" +
  "Include referenced fonts (where license permits)".
- Import: schema-version gate first; then drops assets into `assets/`,
  fonts into `fonts/`, templates into `templates/`. Reports success with
  count of items installed.
